
Approaches and tools used in the project:

1. Requests (module for sending HTTP requests with Python)
2. Config.yaml (file to store some global non-sensitive data)
3. Conftest.py (file for storing fixtures)
4. Requirements.txt (file for denoting all the installed modules)
5. Pytest (framework used for Fixture support and Parameterized testing)
6. Inheritance (to use BaseClass content on each of the child classes)
7. Pydantic (module for validating response schema)
8. Match case statement
9. Allure (for recording and reporting test results outputs)
10. Logging (module for custom logs besides Allure)
11. OS (module to avoid hardcoding file paths)
